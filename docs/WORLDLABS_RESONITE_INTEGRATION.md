## 3. Concrete Integration Path

### Phase A — World Authoring (Ready now)
1. **AI prompt → 3D world**: Use `worldlabs-mcp` tool `generate_world_from_text` with a prompt like `"A busy Vienna coffee shop, wooden tables, afternoon light"`
2. **Download assets**: SPZ (visual) + GLB (collision) land in `%TEMP%\worldlabs\`
3. **Send to Resonite**: `resonite-mcp` OSC export button sends GLB URL to running Resonite headless
4. **Result**: Photorealistic coffee shop environment in Resonite, ready for agents/humans

### Phase B — Robot Avatar Deployment
1. **Avatar model**: Import Unitree G1 URDF → GLB conversion (existing robotics-mcp pipeline)  
   Or use a stylised humanoid stand-in
2. **Spawn via resonite-mcp**: `resonite_inventory_spawn` places avatar in the Marble world session
3. **Wire OSC joint control**: ProtoFlux in-world listens on `/avatar/joint/{name}/rotation`
4. **Drive from ROS2**: `osc-mcp` publishes joint states from ROS2 → Resonite avatar moves

### Phase C — HRI Loop (Full stack)
```
ROS2 Navigation Stack
    │
    ▼ joint states (30 Hz)
osc-mcp → /avatar/joint/* → Resonite → Robot avatar moves
                                │
                           Real humans in session observe
                                │
                     Proximity / audio / collision events
                                │
                         OSC port 9001 responses
                                │
                    osc-mcp recordings buffer
                                │
                    resonite-mcp reads sensor data
                                │
                          → ROS2 behaviour tree
```

### Phase D — robofang Council Integration
The Council of Dozens can orchestrate this entire pipeline as a workflow:

```python
# Example Council task: "Generate and test robot in coffee shop"
council.dispatch(
    task="HRI test in coffee shop",
    agents=[
        "worldlabs-agent",    # generates world
        "resonite-agent",     # deploys to Resonite
        "robotics-agent",     # controls avatar via OSC
        "memops-agent",       # logs results
    ]
)
```

Each agent calls the relevant MCP tool. robofang orchestrates the sequence.

## 5. Comparison: Resonite vs Gazebo for Your Stack

| | Gazebo | Resonite |
|---|---|---|
| Kinematics accuracy | ✅ Full URDF/ODE | ❌ Game physics |
| SLAM / nav testing | ✅ Standard use case | ❌ Wrong tool |
| Real human interaction | ❌ | ✅ Core feature |
| World generation from photos | ❌ | ✅ via worldlabs-mcp |
| Cost per HRI session | High (lab setup) | ~0 (free VR) |
| Remote participants | Hard | Trivial |
| OSC robot control | Via ROS plugins | Native |

**Decision**: Keep Gazebo for physics validation / SLAM. Use Resonite for HRI evaluation, demo scenes, and social navigation testing.
